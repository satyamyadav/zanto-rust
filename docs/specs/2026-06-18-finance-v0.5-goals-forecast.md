# Spec: Finance v0.5 — "Goals & Forecast"

**Date:** 2026-06-18  ·  **Status:** Draft (awaiting decisions — no code yet)
**Source plan:** [docs/product/finance-next-version-plan.md](../product/finance-next-version-plan.md) Phase 4 (final phase).
**Builds on:** v0.2 money model, v0.3 budgets/recurring, v0.4 accounts/net worth.
**Goal:** Make the app forward-looking: savings/debt **goals** with progress, a simple **cash-flow forecast** ("you'll end the month with ~£X"), and **proactive pace nudges** ("dining is on track to blow its budget"). All deterministic Rust, all local.

---

## 1. Scope

**In:**
- **Goals:** savings and debt-payoff goals, progress tracked automatically against an account's balance (Decision Q1).
- **Forecast:** project this month's end net worth / spend from run-rate + known recurring charges (Decision Q2).
- **Proactive pace nudges:** flag a budget category that's on track to exceed before it does (in-app + native, reusing v0.3's `notify`).

**Out:** anything external/live (unchanged). Multi-currency. Investment returns. This is the last planned phase — after it, the roadmap is complete.

---

## 2. Goals (backend)

New store `finance_goals` (latest-wins list, like budgets/accounts):
```jsonc
{ "goals": [ { "name": "Emergency fund", "kind": "savings", "account": "Savings", "target": 10000, "target_date": "2026-12" } ], "saved_at": 17... }
```
- `kind ∈ savings | debt`. Progress is derived from the linked **account balance** (Decision Q1):
  - **savings:** `current = max(0, account.balance)`, `progress = current / target` (clamped 0..1), `remaining = max(0, target − current)`.
  - **debt:** the account is a liability (balance typically negative); `owed = max(0, −account.balance)`; `progress = target ≤ 0 ? (owed == 0 ? 1 : 0) : clamp(1 − owed/target_or_initial)`. For v0.5 keep it simple: `owed` + show "£owed left to pay off" (target = the original debt for the bar; if absent, just show owed).
- `get_goals`, `do_save_goals` (action), query `goals` → `{ goals: [...] }`.
- query/overview includes computed `goal_status: [{ name, kind, account, target, current, owed?, progress, remaining? }]` via a pure helper `compute_goal_status(goals, account_balances)` — unit-tested.

## 3. Forecast (backend)

Pure helper `compute_forecast(records, net_worth, today) -> Value` (deterministic):
- `month_to_date_expense` / `month_to_date_income` = current month sums (reuse normalize_txn, expenses/income only; transfers excluded).
- `avg_monthly_expense` = mean expense over the last **3 complete** months (exclude the current partial month); `avg_monthly_income` likewise.
- `expected_expense = max(month_to_date_expense, avg_monthly_expense)`; `expected_income = max(month_to_date_income, avg_monthly_income)`.
- `projected_net_worth = net_worth + (expected_income − month_to_date_income) − (expected_expense − month_to_date_expense)` (net worth already includes month-to-date flows; add only the *remaining* projected flows).
- Return `{ month, projected_net_worth, expected_income, expected_expense, avg_monthly_expense }`.
- query `forecast`. Also surface `projected_net_worth` in `compute_overview` for a dashboard chip. Unit-tested on synthetic history (3 months at £1000 expense ⇒ avg £1000; projection reflects run-rate).

## 4. Proactive pace nudges (backend + reuse)

Extend `compute_budget_status` (v0.3) with pace, using the day-of-month fraction `f = day_of_month / days_in_month`:
- `projected = f > 0 ? spent / f : spent` (run-rate to month end).
- add fields `projected`, `on_track_to_exceed = !over && projected > limit`.
- `compute_overview` adds `pace_warnings: [{ category, limit, spent, projected }]` = budgeted categories with `on_track_to_exceed`.
- Frontend fires the existing `ipc.notify` once per month per category for new pace warnings (dedupe key `zanto.finance.pace.<YYYY-MM>.<category>`), same pattern as the v0.3 overspend nudge, plus an in-app chip.
- `days_in_month` via a small helper (use the civil-date math already added in v0.3, or month-length lookup with a leap-year check).

## 5. Frontend

`crates/zanto-desktop/src/lib/apps/finance/`:

- **Goals tab** `Goals.svelte`: list `goal_status` as progress bars — savings: "£current / £target ({pct}%)"; debt: "£owed left". Color `text-success` when complete. Net-worth headline already exists (v0.4).
- **GoalsEditor** `GoalsEditor.svelte` (edit mode, like AccountsEditor): rows of `{ name, kind select (savings/debt), account select, target number, target_date? }`; add/remove; Save → `save_goals`.
- **Forecast** surfaced on the dashboard: a card "Projected end of {month}: {formatCurrency(projected_net_worth)}" with a sub-line "≈ {expected_income} in, {expected_expense} out". Also a `forecast` widget kind.
- **Pace nudge** banner: when `overview.pace_warnings.length`, an amber `bg-warning/10 text-warning` chip "Dining is on track to exceed its budget" (+ native notify, deduped).
- **Widgets:** add `goals` (goal progress) and `forecast` widget kinds + sources; add a `projected_net_worth` KPI source. Update `default_widgets` (Rust) to include a Goals widget and a Forecast widget.
- Currency via `formatCurrency`.

## 6. Acceptance criteria
1. A savings goal of £10000 linked to "Savings" (balance £2500) shows 25% progress; reaching £10000 shows complete.
2. A debt goal on "Card" (balance −£800) shows "£800 left to pay off"; at £0 it's complete.
3. With ~£1000/month expense history and £300 spent 10 days into a 30-day month, the forecast projects ~£1000 expense and a month-end net worth reflecting the remaining ~£700.
4. A £200 dining budget with £160 spent by day 18 (f=0.6 ⇒ projected £266) shows a pace warning "on track to exceed" and fires one native nudge for the month.
5. `cargo build` + `cargo test` (goals, forecast, pace) + `pnpm check` + `pnpm build:web` green.

## 8. Resolved decisions (locked 2026-06-18)
- **Q1 → Account-linked goals.** Progress derives from the linked account's balance; no manual contributions. A standalone savings target uses a dedicated Savings account.
- **Q2 → Rest-of-this-month run-rate.** Forecast projects this month's end (net worth + remaining projected income − remaining projected expense). Next-month projection is out of scope.
- **Q3 → In-app + native.** Pace warnings show an amber chip AND fire `ipc.notify` once per month/category (dedupe `zanto.finance.pace.<YYYY-MM>.<category>`), matching the v0.3 overspend nudge.
